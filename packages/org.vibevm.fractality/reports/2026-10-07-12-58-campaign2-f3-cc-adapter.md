# Campaign 2 · Ф3 — адаптер Claude Code: фабрика дотянулась до сессии

_2026-07-10, ~12:38–12:58. Коммит: `4e2c71c` (одно связное feat).
Floor: all green (conform 0, specmap 18/59/0, test-gate xfail-strict)._

## Что сделано

Край харнесса, целиком на Rust-бинаре (закон «никакого python в
продукте» соблюдён буквально — питон Ф0 остался в scratch):

- **`fractality hook <event>`** — session-start (регистрация +
  инъекция живого скорборда + экспорт `FRACTALITY_BOSS_SESSION`
  через `CLAUDE_ENV_FILE`), post-tool-use (WorkTool-ноты с весом
  `duration_ms`), session-end; user-prompt-submit и stop — резолвят
  сессию и молчат до Ф4.
- **`fractality statusline`** — стрип из движка; `frl: mc down` при
  лежащей шине.
- **`fractality harness install|status|remove claude-code`** —
  managed-blocks, адаптированный к JSON: маркер владения — командная
  строка нашего exe; чужие хуки и чужой statusLine неприкосновенны
  (notice, не clobber); битый JSON — жёсткий стоп без авторемонта.
- **`fractality fetch`** — починен дрейф D12 (плейбуки ссылались на
  несуществующий глагол); reqwest получил rustls. Ячейку написал
  turbo-делегат по спецификации, я ревьюнул и повесил один clippy-фикс.
- MT-C2-02 автор-ован (продуктовая форма P2). Живой смоук: полный
  цикл install → hook-и → remove → `{}` → fetch 36 КБ TLS.

## Решения (главное)

- **Закон доступности — абсолют:** любой сбой хука = exit 0 + пустой
  вывод. Сломанная инициатива не имеет права сломать сессию босса.
  Цена: сбои беззвучны (см. висяки).
- **Хуки connect-only, никогда не автостартят демона.** Демон под
  сессией — осознанный акт (mc start / первый CLI-глагол босса), а
  не сюрприз от хука. Плюс ноль спавн-латентности.
- **`session begin` как lookup в каждом хуке** — hook-процесс не
  видит Bash-экспорта, поэтому резолв по `(harness, external_id)`;
  API не расширяли.
- **Ownership = командная строка.** Ни маркер-полей в чужом JSON,
  ни эвристик: детерминированный скан «fractality*-exe + наш глагол».
  Смешанные группы теряют только наши команды.
- **Цель по умолчанию — settings.local.json** (машинный скоуп, RP3);
  `--project` — явный опт-ин в коммитимый файл.
- **Один feat-коммит вместо четырёх плановых** — main.rs прошивает
  все глаголы разом; некомпилирующиеся промежуточные коммиты ради
  плана — фикция. Дрейф заледжерен.

## Идеи и размышления

- Инъекция скорборда на SessionStart — это barkain-BD3 (ленивый
  стаб), но с живыми числами вместо правил. Правила остаются в
  снипете 75; хук несёт только факты.
- reqwest 0.13 переименовал TLS-фичу (rustls-tls → rustls) — ровно
  тот класс дрейфа, который D11 предсказала. Пины окупаются.
- Гейт дважды поймал меня в этой фазе (бюджет main.rs; два expect в
  domain-логике harness.rs) — дисциплина работает как задумано:
  до коммита, не после.

## Косяки и висяки

- **Беззвучность отказов хуков.** По закону доступности сбой =
  тишина; отлаживать «почему не инъецировалось» сегодня можно только
  руками. Нужен opt-in диагностический след (env
  `FRACTALITY_HOOK_DEBUG=file` → строка в лог) — маленький хвост Ф4/Ф6.
- **`FRACTALITY_HOME` и хуки:** hook-процесс наследует env Клода;
  если босс работает со scratch-home (MT-ы!), надо экспортировать
  `FRACTALITY_HOME` в сессию ДО старта — MT-C2-02 так и написан, но
  это wart: продуктовый путь (реальный `~/.fractality`) работает
  из коробки, scratch — с ручкой.
- **`harness install` не проверяет живость демона** — поставит хуки
  и на мёртвую шину (хуки будут no-op). Идея: status подсказывает
  «mc: down — start it».
- **`refreshInterval: 30` захардкожен**; конфиг инициативы
  (`~/.fractality/initiative.toml`, D8) появится в Ф4 — туда же.
- **Stop/UserPromptSubmit пока пустышки** — устанавливаются заранее
  (чтобы Ф4 не трогал settings повторно), но до Ф4 это два холостых
  спавна exe на событие. Осознанный трейд (6 мс каждый).
- Стратегически: **вторая инсталляция (VibeVM Pixel/Codex)** покажет,
  что из adapter-слоя вытащить в общий «harness-SDK»; пока I4
  держится тем, что вся политика в движке.
