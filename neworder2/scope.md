## Scanning hygiene (standing rule for every scan in this run)

Every `find` / `grep` in this run **prunes** these before counting or reading —
they are duplicated, generated, or built, never source:
```
node_modules/  target/  dist/  .git/  .vibe/  vibedeps/  (any depth)  +  packages/**/runs/*.jsonl
```
Canonical package sources are found at **depth-4** (`packages/<group>/<name>/
<version>/`); anything deeper under a package is a materialized-dep copy — skip it.
(Owner correction, this session: an un-pruned count double-counts the same files
many times over.)
