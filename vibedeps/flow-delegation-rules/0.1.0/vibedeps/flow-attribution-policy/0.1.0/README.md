# `flow:attribution-policy` — choose your authorship surface deliberately

A vibevm `flow` package that installs a **deliberate attribution
posture** for an AI-assisted repository. Coding agents attribute
themselves by default — trailers in commits, "generated by" lines —
and a project that never decides ends up with an accidental,
inconsistent record: neither a clean human-authored surface nor a
reliable provenance trail. This package makes the choice explicit
and enforced.

**The default posture is concealment**: the repository presents a
human-authored surface on every artifact. The honest rationale ships
with the rule — some jurisdictions regulate or may regulate machine
authorship of software, and an owner not subject to such regulation
may proactively remove the hook. The open-disclosure posture is
documented as a first-class alternative, and is the *mandatory*
choice where law, employer, or upstream projects require disclosure.
Verify both before adopting the default; the protocol tells you how.

This package ships three pieces of content plus a boot snippet:

- `spec/flows/attribution-policy/ATTRIBUTION-POLICY.md` — the full
  policy: what attribution governs, the concealment posture and its
  honest rationale, the single-place law, how to instruct an agent,
  and the slip drill.
- `spec/flows/attribution-policy/disclosure-alternative.md` — the
  open posture: when it is mandatory, when it is the better fit,
  what it costs, and how to switch postures forward-only.
- `spec/flows/attribution-policy/enforcement-checklist.md` — the
  eight leak surfaces, a pre-push scan, tool configuration, and the
  periodic audit line.
- `spec/boot/55-flow-attribution-policy.md` — boot snippet loaded at
  session start: the rule, the honest reason, the scope carve-outs,
  and the never-do list. Under the single-place law, this snippet is
  the one place in a consuming project where the topic lives.

## Install

```bash
vibe install flow:attribution-policy
```

## Uninstall

```bash
vibe uninstall flow:attribution-policy
```

Uninstalling removes every file the package wrote, including the
boot snippet. User-owned files are never touched.

## Composition

- `flow:atomic-commits` (`30-…`) pins the commit-message format this
  policy scans, and its frozen-history rule governs the
  pushed-slip drill: no history rewrite without owner approval.
- `flow:decision-records` (`25-…`): the posture choice is a recorded
  decision with a revisit trigger tied to regulation changes.
- `flow:wal` (`10-…`): AI-workflow documents like the WAL protocol
  are explicitly carved out — they organize work and never mark
  artifacts as machine-authored.

## Philosophical background

Extracted from *AI-native development*, chapter 2 («Shared state:
файлы как IPC»), the aside on commit metadata: decide consciously
whether commits may say a model made them, and if the answer is no,
tell the model the true reason. The book ships in Russian inside
`flow:redbook` at `spec/book/ru/`.
