# M1.16 — registry redirect smoke

## Purpose

Verify the M1.16 registry-redirect path against real GitHub repos
end to end. Covers `vibe registry redirect <pkgref> --to <url>`
(stub creation), `vibe registry redirect-sync <pkgref>` (target-tag
mirroring), and consumer-side `vibe install <pkgref>` resolving
through the stub transparently (PROP-002 §2.4.2).

The hermetic e2e
(`cli_e2e::install_via_redirect_pass_through_tag` and friends) runs
against `file://` bare repos with a single `[[registry]]`. This
test runs against the real `vibespecs` GitHub org plus an external
`olegchir/...` target so we exercise the `git archive --remote`
→ shallow-clone fall-back and the `inject_token` / `set_remote_url`
plumbing inherited from M1.14.

Run after touching any of: `vibe-cli/src/commands/registry.rs::run_redirect`,
`run_redirect_sync`, `try_fetch_redirect_for_url`,
`MultiRegistryResolver::fetch_manifest`, `follow_redirect`,
`fetch_via_redirect`.

## Preflight

- `git` ≥ 2.31 on `PATH`.
- `vibe` built and available — `cargo build --release -p vibe-cli`
  or `cargo install --path crates/vibe-cli --locked`.
- A GitHub personal access token with `repo` scope on both
  `vibespecstest1` (registry org) and `vibespecstest2` (target
  host), present at `~/.vibe/github.publish.token`.

Test fixtures live in dedicated test orgs so the canonical
`vibespecs` org stays populated with real packages only:

- `vibespecstest1` (GitHub) — registry-side stubs.
- `vibespecstest2` (GitHub) — external-author / target repos.
- `vibespecstest3` (GitVerse) — counterpart for GitVerse-side
  cross-host smoke (used by `cli_live_e2e`, not this recipe).

## Setup — clean slate

```bash
SCRATCH=$(mktemp -d)
export VIBE_REGISTRY_CACHE="$SCRATCH/cache"
mkdir -p "$VIBE_REGISTRY_CACHE"
TARGET_ORG="vibespecstest2"
REGISTRY_ORG="vibespecstest1"
TARGET_NAME="vibevm-m1-smoke-feat-helper"
TARGET_URL="https://github.com/$TARGET_ORG/$TARGET_NAME"
STUB_URL="https://github.com/$REGISTRY_ORG/feat-helper"
TOKEN=$(cat ~/.vibe/github.publish.token | tr -d '\n\r')
```

## Steps

### 1. Create the target repo on GitHub (under `vibespecstest2`).

```bash
cat > "$SCRATCH/create-target.json" <<EOF
{
  "name": "$TARGET_NAME",
  "description": "vibevm M1.16 redirect smoke target — safe to delete",
  "private": false,
  "auto_init": false,
  "has_issues": false,
  "has_projects": false,
  "has_wiki": false
}
EOF

curl -s -o "$SCRATCH/repo-create.json" -w '%{http_code}\n' \
  -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  -H "User-Agent: vibe-smoke-walk" \
  -H "Content-Type: application/json" \
  --data-binary @"$SCRATCH/create-target.json" \
  https://api.github.com/orgs/$TARGET_ORG/repos
```

**Expected:** HTTP `201` (or `422` if the repo already exists from
a prior smoke pass — either delete it first or skip step 2).

### 2. Push a minimal `feat:helper@0.1.0` payload to the target.

```bash
mkdir -p "$SCRATCH/feat-helper-pkg"
cd "$SCRATCH/feat-helper-pkg"
cat > vibe.toml <<'EOF'
[package]
name = "helper"
kind = "feat"
version = "0.1.0"

[writes]
files = [
    "spec/feats/helper/HELPER.md",
]
EOF
mkdir -p spec/feats/helper
echo "# feat:helper — redirect smoke target" > spec/feats/helper/HELPER.md

git init --initial-branch=main -q
git config user.email "smoke@local"
git config user.name "Smoke"
git add -A && git commit -qm "feat:helper@0.1.0"
git tag -a v0.1.0 -m "v0.1.0"
git remote add origin "https://x-access-token:$TOKEN@github.com/$TARGET_ORG/$TARGET_NAME.git"
git push -q -u origin main
git push -q origin v0.1.0
git remote set-url origin "$TARGET_URL"
```

**Expected:** Pushes succeed. `git ls-remote --tags $TARGET_URL`
returns `refs/tags/v0.1.0`.

### 3. Create the redirect stub in the `vibespecstest1` org.

```bash
mkdir -p "$SCRATCH/project-init"
cd "$SCRATCH/project-init"
vibe init --path . --quiet

# `vibe init` writes the canonical `vibespecs` / `vibespecs-gitverse`
# defaults; replace them with the test registry so the redirect
# command targets `vibespecstest1` and not the real org.
cat > vibe.toml <<EOF
[project]
name = "stub-init"
version = "0.0.1"

[[registry]]
name = "$REGISTRY_ORG"
url = "https://github.com/$REGISTRY_ORG"
EOF

vibe registry redirect feat:helper \
  --to "$TARGET_URL" \
  --description "M1.16 redirect smoke; delegated to vibespecstest2; safe to delete" \
  --path .
```

**Expected:**

```
Creating redirect stub: feat:helper → https://github.com/vibespecstest2/vibevm-m1-smoke-feat-helper
  → Loaded publish token from C:\Users\…\github.publish.token (value redacted)
  → Created repository `feat-helper` on `github.com`
  → Pushed stub `vibe-redirect.toml` to `main`

vibe registry redirect: stub `https://github.com/vibespecstest1/feat-helper` delegates `feat:helper` → `https://github.com/vibespecstest2/…`
```

Verify the stub:

```bash
git ls-remote "$STUB_URL"
curl -s "https://raw.githubusercontent.com/$REGISTRY_ORG/feat-helper/main/vibe-redirect.toml"
```

**Expected:** ls-remote shows `main` (no tags yet — sync is the
next step). The marker file echoes:

```toml
[redirect]
target_url = "https://github.com/vibespecstest2/vibevm-m1-smoke-feat-helper"
description = "..."
```

### 4. Mirror the target's tags into the stub.

```bash
cd "$SCRATCH/project-init"
vibe registry redirect-sync feat:helper --path .
```

**Expected:**

```
Syncing target tags into stub: feat:helper
  → Pushed tag `v0.1.0` into stub

vibe registry redirect-sync: pushed 1 tag into stub `feat:helper`. 0 tags were already present.
```

Now the stub has `refs/tags/v0.1.0` pointing at its `main` commit
(the marker-file commit), and the target also has `refs/tags/v0.1.0`
pointing at the package payload. `pass-through-tag` policy is what
ties the two together at install time.

### 5. Install through the redirect from a fresh project.

```bash
mkdir -p "$SCRATCH/project-install"
cd "$SCRATCH/project-install"
vibe init --path . --quiet

# Same registry override as in step 3 — the consumer resolves
# through `vibespecstest1`, not the real `vibespecs` org.
cat > vibe.toml <<EOF
[project]
name = "consume"
version = "0.0.1"

[[registry]]
name = "$REGISTRY_ORG"
url = "https://github.com/$REGISTRY_ORG"
EOF

vibe install feat:helper@^0.1 --path . --assume-yes
```

**Expected:**

```
Resolving 1 root package…

Plan for feat:helper@0.1.0
  create  spec/feats/helper/HELPER.md

  → Installing feat:helper@0.1.0
  ✓ created  spec/feats/helper/HELPER.md

Installed 1 package (1 file written).
```

### 6. Verify lockfile records target_url + via_redirect.

```bash
cat vibe.lock | grep -E 'source_url|via_redirect|source_kind|registry'
```

**Expected:**

```toml
registry = "vibespecstest1"
source_url = "https://github.com/vibespecstest2/vibevm-m1-smoke-feat-helper"
source_ref = "v0.1.0"
source_kind = "registry"
via_redirect = "https://github.com/vibespecstest1/feat-helper.git"
```

The target URL is in `source_url`; the stub URL is in the new
`via_redirect` field. `source_kind = "registry"` (not `git`) — the
package was found through a registry stub, just delegated to an
external content URL.

### 7. Verify token-discipline.

```bash
grep -r "x-access-token" "$VIBE_REGISTRY_CACHE/" 2>/dev/null
find "$VIBE_REGISTRY_CACHE" -name "config" -path "*feat-helper*" \
  | xargs grep -E '^\s*url' 2>/dev/null
```

**Expected:** empty `grep` output (no token bytes in the cache).
The `.git/config` URLs are all plain (no `x-access-token`).

## Cleanup

```bash
# Delete both the target and the stub repos (optional; safe to
# leave for the next smoke pass).
for repo in "$TARGET_ORG/$TARGET_NAME" "$REGISTRY_ORG/feat-helper"; do
  curl -s -X DELETE \
    -H "Authorization: Bearer $TOKEN" \
    -H "Accept: application/vnd.github+json" \
    -H "X-GitHub-Api-Version: 2022-11-28" \
    -H "User-Agent: vibe-smoke-walk" \
    https://api.github.com/repos/$repo
done

rm -rf "$SCRATCH"
unset VIBE_REGISTRY_CACHE TARGET_ORG REGISTRY_ORG TARGET_NAME TARGET_URL STUB_URL TOKEN
```

## What to file if it fails

- Output of step 5 with `VIBE_LOG=debug`.
- `cat vibe.lock` if generated — both `source_url` and `via_redirect`
  fields.
- `find $VIBE_REGISTRY_CACHE -name '*.toml' -path '*feat-helper*'`
  to confirm the marker-file probe and the package-manifest read
  both landed on disk.
- Hash of the GitHub stub at the marker-file commit: `git ls-remote
  https://github.com/vibespecstest1/feat-helper`.
