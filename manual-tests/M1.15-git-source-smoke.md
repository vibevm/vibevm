# M1.15 — git-source dependency smoke

## Purpose

Verify the M1.15 git-source path against a real GitHub repo end to
end. Covers the Cargo / npm / Poetry / Bundler-style affordance —
declaring a dep as `{ git = "https://...", tag = "v0.1.0" }` instead
of resolving it through `[[registry]]`. The hermetic e2e
(`cli_e2e::install_from_git_source_with_tag_records_source_kind_git`)
runs against a `file://` bare repo; this test runs against
`https://github.com/...` so we exercise the `git archive --remote`
→ shallow-clone fall-back the GitHub case forces (PROP-002 §2.4.1,
GitPackageRegistry::fetch_manifest_at_ref two-path read).

Run after touching any of: `--git`/`--tag`/`--branch`/`--rev` flag
parsing in `vibe-cli/src/commands/install.rs`,
`fetch_manifest_at_ref` in `vibe-registry`, the resolver's
git-source dispatch in `MultiRegistryResolver`.

## Preflight

- `git` ≥ 2.31 on `PATH`.
- `vibe` built and available — either `cargo build --release -p
  vibe-cli` or `cargo install --path crates/vibe-cli --locked`.
- A GitHub personal access token with `repo` scope on the
  `vibespecstest2` test org, accessible at
  `~/.vibe/github.publish.token` (one line, file-scoped per
  PROP-000 §20). The token is used to **create** the test target
  repo via `POST /orgs/vibespecstest2/repos`; once created, the
  smoke walk reads the public repo without a token.

Test fixtures live in the dedicated test orgs so the canonical
`vibespecs` org stays populated with real packages only:

- `vibespecstest1` (GitHub) — registry-side stubs and smoke
  packages.
- `vibespecstest2` (GitHub) — external-author / target-side repos.
- `vibespecstest3` (GitVerse) — the GitVerse side of the
  cross-host story.

## Setup — clean slate

```bash
SCRATCH=$(mktemp -d)
export VIBE_REGISTRY_CACHE="$SCRATCH/cache"
mkdir -p "$VIBE_REGISTRY_CACHE"
TARGET_ORG="vibespecstest2"
TARGET_NAME="vibevm-m1-smoke-flow-internal"
TARGET_URL="https://github.com/$TARGET_ORG/$TARGET_NAME"
TOKEN=$(cat ~/.vibe/github.publish.token | tr -d '\n\r')
```

## Steps

### 1. Create the target repo on GitHub.

```bash
cat > "$SCRATCH/create-body.json" <<EOF
{
  "name": "$TARGET_NAME",
  "description": "vibevm M1.15 git-source smoke target — safe to delete",
  "private": false,
  "auto_init": false,
  "has_issues": false,
  "has_projects": false,
  "has_wiki": false
}
EOF

curl -s -o "$SCRATCH/repo-create.json" -w '%{http_code}\n' \
  -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  -H "User-Agent: vibe-smoke-walk" \
  -H "Content-Type: application/json" \
  --data-binary @"$SCRATCH/create-body.json" \
  https://api.github.com/orgs/$TARGET_ORG/repos
```

**Expected:** HTTP `201`. If the repo already exists from a prior
smoke walk, you get `422` — either delete it (`curl -X DELETE …`)
and retry, or reuse it (skip step 2).

### 2. Push a minimal `flow:internal@0.1.0` payload to the target.

```bash
mkdir -p "$SCRATCH/flow-internal-pkg"
cd "$SCRATCH/flow-internal-pkg"
cat > vibe.toml <<'EOF'
[package]
name = "internal"
kind = "flow"
version = "0.1.0"

[writes]
files = [
    "spec/flows/internal/MANIFEST.md",
]
EOF
mkdir -p spec/flows/internal
echo "# flow:internal — git-source smoke" > spec/flows/internal/MANIFEST.md

git init --initial-branch=main -q
git config user.email "$(git config --global user.email || echo 'smoke@local')"
git config user.name "$(git config --global user.name || echo 'Smoke')"
git add -A && git commit -qm "flow:internal@0.1.0"
git tag -a v0.1.0 -m "v0.1.0"
git remote add origin "https://x-access-token:$TOKEN@github.com/$TARGET_ORG/$TARGET_NAME.git"
git push -q -u origin main
git push -q origin v0.1.0
git remote set-url origin "$TARGET_URL"
```

**Expected:** Both pushes succeed. `git ls-remote --tags
https://github.com/$TARGET_ORG/$TARGET_NAME` returns
`refs/tags/v0.1.0` plus its peeled form.

### 3. Install via git-source from a fresh project.

```bash
mkdir -p "$SCRATCH/project"
cd "$SCRATCH/project"
vibe init --path . --quiet
vibe install flow:internal --git "$TARGET_URL" --tag v0.1.0 \
  --path . --assume-yes
```

**Expected:**

```
Resolving 1 root package…

Plan for flow:internal@0.1.0
  create  spec/flows/internal/MANIFEST.md

  → Installing flow:internal@0.1.0
  ✓ created  spec/flows/internal/MANIFEST.md

Installed 1 package (1 file written).
```

### 4. Verify lockfile + manifest provenance.

```bash
cat vibe.toml | grep -A2 'requires.packages'
cat vibe.lock | grep -E 'source_url|source_ref|source_kind'
```

**Expected:**

- `vibe.toml` carries:
  ```toml
  [requires.packages."flow:internal"]
  git = "https://github.com/vibespecstest2/vibevm-m1-smoke-flow-internal"
  tag = "v0.1.0"
  ```
- `vibe.lock` records:
  ```toml
  source_url = "https://github.com/vibespecstest2/vibevm-m1-smoke-flow-internal"
  source_ref = "v0.1.0"
  source_kind = "git"
  ```

### 5. Verify token-discipline invariant on the cache.

```bash
grep -r "x-access-token" "$VIBE_REGISTRY_CACHE/" 2>/dev/null
```

**Expected:** empty output. The git-source clone path does not
embed credentials in `.git/config`.

## Cleanup

```bash
# Delete the GitHub test repo (optional; safe to leave for the
# next smoke pass).
curl -s -X DELETE \
  -H "Authorization: Bearer $TOKEN" \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  -H "User-Agent: vibe-smoke-walk" \
  https://api.github.com/repos/$TARGET_ORG/$TARGET_NAME

rm -rf "$SCRATCH"
unset VIBE_REGISTRY_CACHE TARGET_ORG TARGET_NAME TARGET_URL TOKEN
```

## What to file if it fails

- Output of step 3 with `VIBE_LOG=debug` (or `RUST_LOG=vibe_registry=trace`).
- `cat vibe.lock` if generated — `source_kind` field.
- `find $VIBE_REGISTRY_CACHE -type d -maxdepth 5` for the cache layout.
- Result of `grep -r x-access-token $VIBE_REGISTRY_CACHE` (must stay empty).
