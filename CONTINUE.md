  `~/.config/vibe/config.toml`, L1 — **узкий**: только `[env]` + `[install].slot_integrity`).
- **L3 (user-project) ОТСУТСТВУЕТ.** Spec §9.5 (`VIBEVM-SPEC.md:1072`) объявляет 5-уровневую
  precedence, но merge-алгоритма НЕТ — только `promote_user_config_env` (`main.rs:246`, env-fallback).
- **`vibe show config`** (`commands/show/config.rs:20`) показывает слои с provenance, но НЕ composite.
- **Precedent resolver'а:** `vibe-publish/src/token.rs:134` `load_token_for_host` — 5-уровневый
  host-aware resolver с `TokenSource` provenance — готовая модель.

### Web pain-points (✓ research) — топ-10 design imperatives (кратко)
1. Layer-order = law (`policy > user-machine > repo-shared > user-project > default`).
2. `vibe config --show-origins` (источник каждого значения, file:line).
3. Array **replace** by default, merge opt-in per-key with declared strategy (dotnet #118204 ловушка).
4. Schema-first TOML; unknown/deprecated → loud warning at boot.
5. Three files, three roles: `vibe.toml` (committed) / `vibe.local.toml` (gitignored) /
   `~/.config/vibe/machine.toml`.
6. `.gitignore` для `*.local.toml` auto-generated by `vibe init`.
7. Secrets out of committed config (`env:` / keychain / local-file); pre-commit refusal.
8. Deprecation = first-class migration (`replaced_by` + `vibe config migrate`).
9. Declarative `applies = live | reload | restart` per setting.
10. CLI flag **layers**, never replaces; cascade shadows warned.
Полный отчёт с первоисточниками — в transcript research-агента; перенос в research-doc при synthesis.
